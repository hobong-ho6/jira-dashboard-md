# 09 — Ticket Cards (요구 B3)

목적: 간트 아래에, 라벨 그룹으로 묶인 티켓들을 카드 형태로 보여주고 그룹을 접고 편다.

## 레이아웃
- 간트(`06`) 아래 별도 섹션. 라벨 그룹(`05`)별로:
  - 그룹 헤더(접기/펴기, `group:{name}` 상태 공유) → 카드 그리드/리스트.
- 카드 그리드는 반응형(예: `repeat(auto-fill, minmax(280px, 1fr))`).

## 카드 구성요소
- 상단: `KEY` + 상태 pill(상태 카테고리 색) + 우선순위 아이콘.
- 본문: `summary`(2줄 말줄임).
- 상위 티켓 줄(sub-task 등 `parent` 있을 때만): `↳ 상위 {부모KEY} {부모summary}`. 클릭 시 부모가 snapshot에 있으면 상세로 이동, 없으면 Jira로 연다(카드 본문 클릭과 분리, stopPropagation). `parent`는 `{key, summary}` 객체(`03`).
- 메타 행: Due 칩(bucket 색 + 날짜, `06`), 담당자 아바타/이름.
- 라벨 칩들(`05` 색).
- Description 링크 칩들(`08`).
- 관계 칩들(`07`) — 공간 절약 위해 개수만 보이고 hover/상세에서 펼침도 가능.
- 카드 클릭(메타/칩 외 영역) → 상세 패널 열기(`10`).

## 인터랙션
- 카드의 칩 클릭은 카드 클릭과 분리(stopPropagation): 링크 칩=새 탭, 관계 칩=이동, 라벨 칩=필터 토글.
- 빠른 액션(선택): 카드 우상단 `⋯` → 상태 변경/Due 변경 단축. 누르면 상세와 동일하게 큐잉(`11`).
- 정렬: 그룹 내 기본 `bucket(overdue→today→thisWeek→later→none)` 후 duedate 오름차순, 동률 priority.

## 상태 동기화
- snapshot 폴링(`12`)으로 카드가 최신화. mutation 후 변경 이슈는 잠깐 "업데이트됨" 하이라이트.

## Definition of Done
- 라벨 그룹별 카드가 접기/펴기 되고 간트와 상태 공유.
- 카드에 상태·Due·담당자·라벨·링크·관계가 모두 표시.
- 카드 클릭으로 상세가 열리고, 칩 클릭은 각자 동작(이벤트 충돌 없음).
- 그룹 내 정렬이 결정적.
