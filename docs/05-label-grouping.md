# 05 — Label Grouping (요구 A1)

목적: 이슈를 라벨로 그룹핑해 라벨별로 관리한다. 라벨 그룹은 간트(`06`)와 카드(`09`) 양쪽의 1차 묶음 단위다.

## 데이터
- `snapshot.labelGroups[]` (`03`/`04`에서 생성). 각 항목: `{ name, count, issueKeys[] }`.
- 한 이슈가 여러 라벨을 가지면 **여러 그룹에 중복 등장**한다(이게 의도된 동작). 카드/간트 막대는 그룹마다 렌더되며, 변경 액션은 항상 `issueKey` 기준이라 어느 그룹에서 눌러도 동일 이슈를 가리킨다.
- 라벨 없는 이슈 → `"(no label)"` 그룹.

## 정렬·표시 규칙
1. 그룹 순서(두 단계):
   - **기본(시드):** `normalize.py`가 `snapshot.labelGroups`를 만들 때 `config.labelOrder`에 있는 라벨을 그 순서대로 먼저, 나머지는 `count` 내림차순, 동률은 이름 오름차순. `"(no label)"`은 항상 맨 끝.
   - **사용자 조정(보기 설정):** "그룹 순서 조정" 모달에서 드래그한 순서는 **로컬 보기 설정**이다. 브라우저가 `data/ui-state.json`(서버 `POST /api/ui-state`, `12`·`13`)의 `groupOrder`에 저장하고, 렌더 시 그 순서를 시드 위에 적용한다(`util.applyGroupOrder`). Jira·Claude `process`와 무관하게 **즉시 반영**된다. `"(no label)"`은 여기서도 항상 맨 끝.
2. 그룹 헤더: `라벨명 · {count}` + 접기/펴기 토글. 색상은 라벨 문자열 해시 → 팔레트(결정적, `12`의 토큰).
3. 그룹 접힘 상태는 간트·카드가 공유한다(같은 라벨을 두 곳에서 동시에 접고 펼 수 있게 상태 키를 `group:{name}`으로 공유).

## 필터(보너스, 1차에 권장)
- 라벨 멀티선택 필터: 선택된 라벨 그룹만 표시.
- 상태 카테고리 필터(new/indeterminate/done), 담당자 필터.
- 필터는 클라이언트 측에서 snapshot을 필터링(추가 Jira 호출 없음).

## 라벨 편집(쓰기, 선택)
- 카드/상세에서 라벨 추가·제거 → `set_labels` 명령으로 큐잉(`03`/`11`). 전체 라벨 배열을 보내 덮어쓴다(부분 patch 아님).

## 라벨 숨김(로컬 보기 설정)
- 특정 라벨 그룹을 대시보드에서 감춘다. **순수 로컬 보기 설정**(`ui-state.hiddenLabels`, `03`)이며 Jira·큐와 무관하다(그룹 순서·접힘과 동일 성격).
- 진입점: (a) 카드 그룹 헤더의 **"숨김"** 버튼(즉시 숨김), (b) **"그룹 순서 조정"** 모달의 라벨별 **표시/숨김 토글**(숨긴 라벨도 목록에 남아 복원 가능).
- 숨긴 라벨은 `app.js`의 `visibleGroups()`·`todayGroups()`에서 제외되어 **간트·카드·오늘 마감** 모두에서 빠진다. 상태줄 이슈 수 등 전체 집계에는 영향 없다.
- `state.js`의 `toggleLabelHidden`/`setHiddenLabels`가 `ui-state.json`에 즉시 영속(서버 `process` 불필요).

## Definition of Done
- 모든 이슈가 최소 한 그룹에 속한다(없으면 `(no label)`).
- 다중 라벨 이슈가 각 라벨 그룹에 나타난다.
- 그룹 순서가 결정적이다: 기본은 `config.labelOrder` 시드를 존중하고, 사용자가 조정하면 `ui-state.groupOrder`가 그 위에 적용된다.
- 모달에서 드래그→저장 시 간트·카드 양쪽이 즉시(서버 `process` 없이) 새 순서로 갱신된다.
- 그룹 접힘 상태가 간트와 카드에서 일관된다.
