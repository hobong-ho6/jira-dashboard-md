# service-spec.md — Jira MCP Dashboard 사용 매뉴얼 (정적 사이트)

> DropWeb 배포용 정적 사이트. dropweb-guide.md 규칙(상대경로, 루트 index.html, CDN 허용) 준수.

## 서비스 개요
- **서비스명:** Jira MCP Dashboard 사용 매뉴얼
- **목적:** 다른 사용자가 대시보드의 기능과 사용법을 빠르게 익히도록 시각 가이드 제공
- **타겟:** 대시보드를 처음 쓰는 팀원(비개발자 포함)
- **핵심 가치:** 실제 사용 예시(이 대시보드를 운영하며 한 작업들)로 설명 — 추상 설명이 아닌 "해본 것" 중심

## 페이지 구성 (사이트맵)
- 단일 페이지 `index.html` (앵커 기반 하이퍼텍스트 네비게이션)
  - `#intro` 대시보드란 / `#install` 프로젝트 받기(클론·실행) / `#start` 쿼리로 시작 / `#views` 화면 구성 / `#detail` 상세·코멘트·링크
  - `#links` 링크 자동 분류(도메인별 칩) / `#customize` 보기 맞춤(영역/그룹 순서·접힘 저장) / `#mutate` 상태·마감일·코멘트
  - `#create` 티켓 생성 / `#slack` Slack 스레드→티켓(핵심) / `#how` 동작 원리 / `#ops` 자동 운영(서버·워쳐·`/loop` 정기 재조회) / `#faq`

## 페이지별 상세 기능
- **사이드바 네비**: 고정, 스크롤스파이로 현재 섹션 강조. 모바일은 상단바 ☰ 토글.
- **앱 목업(다크)**: 실제 대시보드를 CSS로 재현(JQL 바·미니 간트·라벨 그룹 카드·영역 ▲▼). 이미지 파일 없이 HTML/CSS로 렌더 → 상대경로 의존성 0.
- **실제 예시 콜아웃**: 실제 처리한 작업(일괄 Closed, 마감일, 생성, Slack→티켓)을 인용.
- **코드 블록 복사 버튼**: script.js가 `<pre>`에 자동 부착.
- **FAQ**: `<details>` 아코디언.

## 데이터 구조
- 정적. 외부 데이터 입력/저장 없음(폼·API 호출 없음).
- 표시 콘텐츠는 하드코딩된 설명/예시.

## 디자인 가이드
- 라이트 테마(가독성) + 대시보드 버킷 색(overdue #e5484d, today #f5a623, week #3b82f6, later #8b8f98) 강조.
- 폰트: Pretendard(CDN), 모노는 시스템 모노.
- 반응형: 880px 이하에서 사이드바 → 상단 햄버거. 720px 이하 그리드 1열.

## 외부 연동
- CDN: Pretendard 웹폰트(jsdelivr). 그 외 외부 호출 없음.

## 파일 구성
- `index.html` (루트) · `style.css` · `script.js` · `service-spec.md`
- 모든 참조 상대경로. 홈 링크 없음(단일 페이지, 앵커).
