흩어진 Jira 이슈를
한 화면에서 보고, 바로 정리하세요.

Jira MCP Dashboard는 내 Jira 이슈를 마감일 타임라인과 라벨 그룹으로 한눈에 보여주고, 상태 변경·마감일·코멘트·티켓 생성을 대시보드에서 바로 처리합니다. 심지어 Slack 스레드 링크 하나로 회의 내용을 요약해 티켓까지 만듭니다.

사용법 시작 → ★ Slack → 티켓 보기
소개

대시보드란 무엇인가요?

Jira에 있는 내 이슈를 읽어와 보기 좋게 시각화하고, 화면에서 만든 변경을 다시 Jira에 반영하는 도구입니다. 브라우저는 보기와 의도 입력만 담당하고, Jira에 대한 실제 읽기·쓰기는 뒤에서 안전하게 처리됩니다.

1단계 · 터미널 열고 저장소 받기 (클론)

터미널 앱 열기

키보드에서 ⌘ + Space 를 누르고 "터미널"(또는 Terminal) 입력 → Enter. (또는 응용 프로그램 ▸ 유틸리티 ▸ 터미널)

아래 명령을 복사해 터미널에 붙여넣기(⌘+V) → Enter

터미널은 기본으로 홈 폴더(예: /Users/내계정)에서 열립니다. 그대로 붙여넣으면 홈 폴더 안에 jira-dashboard-md 폴더가 생기고 그 안으로 들어갑니다. (오른쪽 위 복사 버튼을 쓰세요.)

git clone https://github.com/hobong-ho6/jira-dashboard-md.git
cd jira-dashboard-md

📁 특정 폴더에 받고 싶다면: Finder에서 그 폴더를 우클릭 → "폴더에서 새로운 터미널 열기" 한 뒤 위 명령을 붙여넣으세요.

⚙️ 처음 git을 쓰면 "명령어 도구 설치" 팝업이 뜰 수 있습니다 — 설치를 누르고, 끝나면 위 명령을 다시 붙여넣으면 됩니다.

이미 받았다면 · 최신 버전으로 업데이트 (git pull) 🔄 NEW

대시보드는 계속 개선됩니다(요일 표시·라벨 편집/숨김·자동 운영 등). 이미 클론한 폴더가 있다면 다시 클론하지 말고, 그 폴더에서 git pull로 최신 코드만 받으면 됩니다.

① 쿼리(JQL)로 시작하기

대시보드의 출발점은 JQL 쿼리입니다. 상단 JQL 바에 "어떤 이슈를 볼지" 입력하고 조회 시작을 누르면, 그 조건의 이슈들이 타임라인과 카드로 채워집니다.

Jira MCP Dashboard
JQLproject in (W3P, UNIFY) AND status in (Open, "In Progress", Reopened) AND resolution = Unresolved AND (assignee in (currentUser()) OR reporter in (currentUser())) ORDER BY priority DESC, updated DESC조회 시작

입력하는 법

카드 그룹 머리글의 "🙈 숨김" 버튼

복원·일괄 관리는 "그룹 순서 조정"에서

순서 조정 모달에는 숨긴 라벨도 남아 있고, 각 라벨의 👁 표시 / 🙈 숨김 토글로 다시 보이게 하거나 여러 개를 한 번에 정리할 수 있습니다.

접기 / 펼치기 동기화 + 저장

그룹을 접으면 타임라인과 카드가 함께 접힙니다(한쪽에서 접으면 양쪽 반영). 그리고 접힌 상태는 저장되어 다음에 열 때 복원됩니다.

실제 예시 카드에서 GuideKim 그룹을 접자 타임라인에서도 같이 접혔고, 새로고침 후에도 접힘·영역 순서가 그대로 복원됐습니다.
작업

⑤ 상태 · 마감일 · 코멘트 변경

상세 패널/카드에서 상태를 바꾸거나, 마감일을 지정하거나, 코멘트를 답니다. 누른 변경은 Jira에 반영됩니다.

코멘트를 답니다. 같은 내용 중복 게시는 자동으로 막습니다.

⑧ 어떻게 동작하나요?

브라우저는 Jira를 직접 호출하지 않습니다. 화면에서 만든 "의도"는 대기열(큐)에 쌓이고, 뒤에서 그 큐를 처리하는 쪽이 Jira에 안전하게 반영한 뒤 화면을 갱신합니다.

→

→
화면 갱신폴링으로 결과 반영

운영 · 자동화

⑨ 자동 운영 — 서버·워쳐·정기 재조회 ⏰ NEW

대시보드는 로컬 서버와 큐 워쳐로 돌아갑니다. 서버는 화면을 띄우고, 워쳐는 여러분이 대시보드에서 누른 변경(상태·마감일·코멘트·라벨 등)을 Claude Code가 Jira에 자동 반영하도록 이어줍니다. (둘 다 Jira를 직접 바꾸지 않습니다 — ⑧ 참고.)

🖥️

서버 = 상시

대시보드 화면을 띄우는 로컬 서버. 세션이 꺼져도 계속 살아 있게 데몬으로 띄우고, 세션 재개 시 자동 복구됩니다.

👀

워쳐 = 세션 동안만

큐에 쌓인 변경을 감지해 Claude Code가 처리하게 잇는 역할. Claude Code 세션이 살아있는 동안만 자동으로 돕니다.

워쳐를 계속 돌리려면 — /loop ⏰

워쳐(자동 처리)와 정기 전체 재조회는 Claude Code 세션이 깨어 있을 때만 유지됩니다. 그래서 주기적으로 자동 실행하려면 Claude Code에 /loop을 걸어 둡니다. 매시간 전체 재조회 + 서버·워쳐 헬스체크를 하는 hourly-resync 스킬을 이렇게 예약합니다:

/loop 1h hourly-resync 스킬 절차대로 전체 재조회 + 서버/워쳐 헬스체크 해줘
Claude Code
매시간 저장된 쿼리로 전체 재조회하고, 서버·워쳐가 꺼져 있으면 되살리도록 예약했습니다. ⏰
무엇을 하나

저장된 JQL로 전체 티켓을 다시 조회해 보드를 최신화하고, 로컬 서버·큐 워쳐가 꺼져 있으면 자동으로 재기동합니다.

왜 필요한가

세션이 살아있는 동안만 자동 처리가 되므로, /loop으로 주기 실행을 걸어 두면 "동기화 / 큐 처리"를 매번 직접 시키지 않아도 최신 상태가 유지됩니다.

⏰ /loop 예약은 그 Claude Code 세션이 살아있는 동안만 유지됩니다. 세션이 완전히 끊기면 다시 /loop 1h …로 걸어 주세요. (서버·워쳐 자체는 세션 재개 시 자동 복구되지만, /loop 예약까지 복구되지는 않습니다.)
실제 예시 매시간 hourly-resync를 /loop 1h로 예약해, JQL 전체 재조회로 보드를 갱신하고 서버·워쳐가 죽어 있으면 자동 재기동하도록 운영했습니다.
도움말

자주 묻는 질문

이슈가 안 보여요.

JQL 바에 쿼리를 넣고 "조회 시작"을 눌렀는지 확인하세요. 쿼리가 없으면 대시보드는 비어 있습니다.

마감일 없는 티켓은 어디 있나요?

"범위 밖 티켓" 영역에 모입니다. 타임라인 7일 구간(D-1~D+5)을 벗어난 것도 여기로 갑니다.

Slack 비공개 채널 스레드도 되나요?

연결된 Slack 계정이 그 채널에 들어가 있어야 읽을 수 있습니다. 미가입 비공개 채널은 접근이 막혀 처리되지 않습니다.

변경이 화면에 늦게 반영돼요.

변경은 큐에 담겨 순차 처리된 뒤 폴링으로 반영됩니다. 잠시 후 자동 갱신되며, 새로고침으로 즉시 확인할 수도 있습니다.

접기·순서 설정이 사라지나요?

아니요. 접힘 상태와 영역/그룹 순서는 저장되어 다음에 열 때 복원됩니다.
